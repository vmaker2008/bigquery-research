[← 목차로 돌아가기](index.md)

## 요금 구조 개요

BigQuery의 비용은 크게 두 가지로 구분됩니다.

```
BigQuery 비용
├── 분석 비용 (쿼리 실행)
│   ├── 온디맨드 (주문형): 처리 데이터 TB당 과금
│   └── 용량 기반 (슬롯): 선불 컴퓨팅 자원 예약
└── 스토리지 비용
    ├── 활성 스토리지 (최근 90일)
    └── 장기 스토리지 (90일 이후, 자동 할인)
```

---

## 1. 온디맨드 (주문형) 요금제

사용한 만큼만 지불하는 방식입니다.

### 가격
- **$6.25 / TiB** (처리된 데이터 기준)
- **매월 1TB 무료** 쿼리 제공

### 특징
- 쿼리 실행 시 스캔된 데이터 양에 비례하여 청구
- 서버나 슬롯 예약 불필요
- 소규모 또는 불규칙적 워크로드에 적합

### 비용 예측 예시
```
10TB 데이터 쿼리 실행 시:
10 TiB × $6.25 = $62.50

단, SELECT * 대신 필요한 열만 선택하면:
필요 열만 스캔 → 실제 비용 대폭 절감
```

> **주의**: `SELECT *` 사용 시 전체 데이터 스캔 비용 발생. 필요한 컬럼만 명시하세요.

## 2. 용량 기반 (슬롯 예약) 요금제

컴퓨팅 자원(슬롯)을 미리 예약하는 방식입니다.

### BigQuery Editions별 가격

| Edition | 특징 | 추천 대상 |
|---------|------|----------|
| **Standard** | 기본 기능, 1년/3년 약정 없음 | 소규모, 탐색적 분석 |
| **Enterprise** | 고급 기능 + SLA | 중/대규모 기업 |
| **Enterprise Plus** | 최고 가용성, 크로스 리전 | 미션 크리티컬 워크로드 |

### 슬롯 옵션
- **자동 확장 슬롯**: 필요에 따라 동적 추가 용량 제공
- **기본 슬롯**: 고정 슬롯 예약
- **월간/연간**: 장기 약정 시 추가 할인

### 장점
- 예측 가능한 고정 비용
- 대규모 지속적 워크로드에 비용 효율적
- 슬롯 우선순위 및 예약 관리 가능

## 3. 스토리지 요금

### 가격 체계

| 유형 | 조건 | 가격 (근사치) |
|------|------|--------------|
| **활성 스토리지** | 최근 90일 이내 수정된 데이터 | ~$0.02 / GB / 월 |
| **장기 스토리지** | 90일 이상 미수정 데이터 | ~$0.01 / GB / 월 (50% 할인) |

### 무료 스토리지
- 매월 **10GB** 무료 스토리지 제공

### 장기 스토리지 자동 전환
- 90일 연속으로 수정되지 않은 테이블 파티션은 자동으로 장기 스토리지 요금 적용
- 데이터 이동 불필요, 자동 적용

## 4. 기타 비용 요소

| 서비스 | 과금 기준 |
|--------|----------|
| **BigQuery ML** | 모델 학습 시 처리 데이터 TB당 과금 |
| **BI Engine** | 예약된 메모리 GB당 시간 과금 |
| **Data Transfer Service** | 일부 데이터 소스는 무료, 타사는 별도 과금 |
| **Streaming Insert** | 삽입된 데이터 GB당 과금 |
| **Storage API** | 읽기 API 사용 시 GB당 과금 |

## 5. 비용 절감 전략

### 쿼리 비용 절감
```sql
-- ❌ 비효율: 전체 컬럼 스캔
SELECT * FROM `project.dataset.table`;

-- ✅ 효율: 필요한 컬럼만 선택
SELECT user_id, purchase_date, amount
FROM `project.dataset.table`;
```

### 파티션으로 스캔 최소화
```sql
-- ❌ 전체 테이블 스캔
SELECT * FROM `orders`;

-- ✅ 파티션 필터 적용
SELECT * FROM `orders`
WHERE DATE(order_date) = '2024-03-01';
-- 해당 날짜 파티션만 스캔!
```

### 비용 예측 도구
- **쿼리 드라이 런**: 실행 전 스캔 데이터 양 확인
- **BigQuery Pricing Calculator**: 예상 비용 계산
- **Cloud Billing 알림**: 예산 초과 시 알림 설정

## 6. 온디맨드 vs 슬롯 예약 선택 기준

```
온디맨드 선택:                    슬롯 예약 선택:
━━━━━━━━━━━━━━━━                 ━━━━━━━━━━━━━━━━━━
✓ 불규칙한 쿼리 패턴              ✓ 일관된 대규모 워크로드
✓ 탐색적 분석                    ✓ 예측 가능한 비용 필요
✓ 월 쿼리 비용이 낮음             ✓ 고성능 SLA 필요
✓ 빠른 시작이 필요한 경우          ✓ 많은 동시 사용자
```
